# Humanoid MJX PD-Control Project

本プロジェクトは、Windows上のWSL2 (Ubuntu) 環境で動作する高速物理エンジン「MuJoCo MJX」を用い、強化学習によって人型ロボット（ヒューマノイド）の姿勢制御と歩行を学習させる実験プロジェクトです。電気電子エンジニアの視点から、最終的には制御による消費電力の解析や、回路設計へのフィードバックを目的としています。

## 🖥️ 実行環境の詳細
本プロジェクトは、以下のOSおよびパッケージ管理環境を前提として設計されています。
- **OS**: Ubuntu (WSL2 / Windows 11)
- **パッケージ管理**: Conda (Miniconda または Anaconda)
  - 強化学習に必要なJAX、CUDA関連ライブラリ、およびMuJoCo MJXは、専用の仮想環境（`mjx_env`）内で管理されます。

## 📜 プロジェクトの成り立ち
本プロジェクトは、CPU駆動であった前身プロジェクト [onion3630/humanoid_battery_sim](https://github.com/onion3630/humanoid_battery_sim) からの派生・進化版です。
従来のCPUベースの低速な演算環境から、Google MuJoCo MJXによるGPUネイティブな並列物理演算環境へと移行し、数千万ステップ規模の高速な学習を実現しています。

## 🐣 初心者のための強化学習（RL）入門
このプロジェクトでは「PPO」というアルゴリズムを使用しています。
- **エージェント（ロボット）**: 関節を動かす指示を出す「脳」にあたります。
- **環境（シミュレーター）**: 物理法則（重力、摩擦、関節の硬さ）を計算します。
- **報酬（スコア）**: 上手く立てたら加点、転んだら減点します。
ロボットは、数千万回の試行錯誤を通じて「どう動けばスコア（報酬）を最大化できるか」を自ら学習します。

## ⚙️ 主要パラメータの解説
各ファイルの先頭にある変数（パラメータ）を変更することで、ロボットの性格や学習の進み方を変えることができます。

### 1. `pd_env.py` (物理・制御パラメータ)
| 変数名 | 意味 | 入力制限 / 目安 |
| :--- | :--- | :--- |
| `KP_GAIN` | 比例ゲイン。関節を目標角度に引き戻すバネの強さ。 | 30.0 〜 100.0 |
| `KD_GAIN` | 微分ゲイン。関節の動きにブレーキをかける粘り気。 | 0.5 〜 10.0 (KPの10%〜20%程度を推奨) |
| `ACTION_LIMIT` | AIが出力する目標角度の最大可動域。 | 0.1 〜 1.0 (1.0で約57度) |
| `FALL_HEIGHT` | 転倒とみなす高さのしきい値。 | 0.5 〜 0.8m |
| `FLY_HEIGHT` | 異常な浮き上がりを検知して強制終了する高さ。 | 2.0m 前後 (報酬ハック対策) |
| `STILL_THRESHOLD` | 静止とみなす全関節速度（L2ノルム）のしきい値。 | 0.05 程度 |
| `STILL_SUCCESS_STEPS` | ゴールとみなす「静止状態の継続ステップ数」。 | 50.0 程度 |
| `SUCCESS_BONUS` | 目標ステップ数静止を維持できた際に与えられるボーナス報酬。 | 500.0 程度 |

### 2. `train_pd.py` (学習パラメータ)
| 変数名 | 意味 | 入力制限 / 目安 |
| :--- | :--- | :--- |
| `TOTAL_TIMESTEPS` | 学習の目標ステップ数。 | 学習器内部のループ仕様に合わせて自動補正されるため任意の数値を指定可能。 |
| `LEARNING_RATE` | 学習率。AIの知識を更新する歩幅。 | 1e-5 〜 1e-4 |
| `NUM_ENVS` | 並列実行するロボットの数。 | 256 〜 4096 (VRAM容量に依存) |
| `NUM_MINIBATCHES` | 1回のアップデートで分割処理するバッチ数。 | 32 程度 |

## 🚀 実行方法
以下の手順は、Ubuntuターミナル上でConda環境がアクティベートされていることを前提としています。

1. **環境の有効化**
   ```bash
   conda activate mjx_env
   ```

2. **学習の開始**
   ```bash
   python train_pd.py
   ```
   - ※ 注意：実行環境のGPU性能によりますが、学習完了までには数十分〜1時間程度を要します。
   - **※ 学習ステップ数の同期について**：Brax内部の `action_repeat` によるループ仕様のズレを防ぐため、プログラム開始時に算出・表示される「最終ステップ数」に合わせて、学習器（PPO）の内部ループ回数が逆算・補正されます。これにより、事前の宣言通りに学習が正確に終了します。

3. **結果の視覚化（MP4出力）**
   学習終了後、生成されたパラメータファイル（.pkg）を引数に指定して実行します。
   ```bash
   python viz_pd.py humanoid_pd_params_YYYYMMDD_HHMMSS.pkg
   ```
   - ※ 実行後、カレントディレクトリに読み込んだモデルと同名の動画ファイル（例：`humanoid_pd_params_YYYYMMDD_HHMMSS.mp4`）が自動生成されます。

## 🎨 ロボットの色の変更方法
MP4動画に出力されるロボットの色を変更したい場合は、`viz_pd.py` 内のレンダラー初期化直前に色を指定するコードを挿入します。

1. `viz_pd.py` を開き、`renderer = mujoco.Renderer(...)` の直前を探します。
2. 以下のように `mj_model.geom_rgba` を書き換えるコードを追加します。

```python
# viz_pd.py 内の例
mj_model = viz_env.sys.mj_model

# すべてのパーツの色を一括変更する場合 (RGBA: 赤, 緑, 青, 不透明度)
# 例：落ち着いた青色にする場合
mj_model.geom_rgba[:] = [0.2, 0.2, 0.8, 1.0] 

renderer = mujoco.Renderer(mj_model, height=480, width=640)
```

## 📊 ログ（進捗表示）の読み方
学習を実行すると、ターミナルへの画面表示と同時に、カレントディレクトリにCSV形式のログファイル（`_trainlog_[日時].csv`）が生成されます。

- **CSVヘッダー**: 1行目から数行にわたり、その学習で使用されたファイル名、各種ハイパーパラメータ、報酬の重み設定などが `#` 始まりで記録されます。
- **Step**: AIが経験した累計のステップ数です。
- **報酬 (Reward)**: 現在のスコアの平均値です。高いほど「上手」に動けています。
- **生存長 (Length)**: 転倒せずに何ステップ耐えられたかを示します。
- **経過 (Time)**: 学習開始から現在までの経過時間（秒）です。

## 🗺️ 今後のロードマップ
### Phase 1: 姿勢維持（ロバスト制御・外乱抑制）
- **目標**: 外部から押されたり、高所から落下しても直立姿勢に復帰・維持する制御の獲得。
- **実装案**: 「頭部の高さ」と「胴体の垂直度」を評価する報酬関数へ再設計。エピソード開始時の高度と姿勢のランダム化（Domain Randomization）を導入。

### Phase 2: 荷物運搬（重心変動への適応）
- **目標**: 両手に荷物（質量オブジェクト）を持った状態でのバランス制御。
- **実装案**: MJCFファイル（XML）を改修し手首にウェイトを追加。カリキュラム学習を用いて徐々に重量を増やす。

### Phase 3: 目的地点への移動（ナビゲーション）
- **目標**: 指定したXY座標へ自律的に歩き出し到達する機能の実装。
- **実装案**: 観測データに目標地点との相対距離と角度を追加。目標に近づくほど高くなる距離報酬を設定。

### Phase 4: 消費電力解析と回路設計（ハードウェア連携）
- **目標**: シミュレーション挙動から電気的仕様を割り出し、電源回路設計へフィードバック。
- **実装案**: 関節の仕事率 $P = \tau \cdot \omega$ から電流波形を推定。データをCSV化してLTspice等の回路シミュレータで検証。
